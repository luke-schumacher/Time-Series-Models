================================================================================
DATA STRUCTURE
================================================================================

Each segmented file contains:

Key Columns:
  - entity_type: 0 = REAL_PATIENT, 1 = PSEUDO_PATIENT_PAUSE
  - sourceID: Scan type (e.g., MRI_MSR_104) or "IDLE" for pseudo-patients
  - segment_index: Which segment in the original sequence
  - original_sequence_id: Link back to original patient sequence
  - pause_duration_seconds: Duration of pause (only for pseudo-patients)

Entity Type Distribution:
  - Real Patient (0): ~84%
  - Pseudo-Patient (1): ~16%
